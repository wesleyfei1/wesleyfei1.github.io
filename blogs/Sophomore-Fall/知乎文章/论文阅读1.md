## Attention结构

### Query,Key,Value

attention这个东西结构非常的简单，也有非常多的变种，但是他们都是由Q(Query),K(Key),V(value)几个东西组成

- Q:可以认为是对于每一个输入token的思考
- K:将每一个token的语义信息进行输入
- V:将Q,K进行信息整合之后得到对于整个句子的理解之后，将**对于整个句子的思考**，转换成对于**句子相关的value**的理解。

而我们的实现方式也非常简单，直接将我们的输入$x=[x_1,x_2,...x_n]$直接乘上一个矩阵

$Query,Key,Value=QW_i^Q,W_i^K,V W_i^V$

### 单头attention

原本的Transformer模型中，我们的方法是**将Q,K进行整合，然后使用softmax放大其中的最重要的token，最后加上value信息**

将Q,K进行整合，google团队使用了最简单的矩阵乘法，因此理论上我们的attention应该是

$Attention(Q,K,V)=softmax(QK^T)V$

但是实际上，如果对于Q有$E(Q_{ij})=0，Var(Q_{IJ})=1$,那么就有对于$QK^T$中的每个元素的方差为$d_K$而这在经过softmax放大之后会导致**完全输出某一个token的信息**，因此为了减小这点，我们的attention(Q,K,V)

$Attention(Q,K,V)=softmax(\dfrac{QK^T}{\sqrt{d_k}})V$

![image-20251015140616191](D:\wesleyfei1.github.io\blogs\Sophomore-Fall\知乎文章\论文阅读1.assets\image-20251015140616191.png)

使用pytorch代码进行实现就是

```python
class Head(nn.Module):
    def __init__(self,head_size):
        super().__init__()
        self.key=nn.Linear(n_embd,head_size,bias=False)
        self.query=nn.Linear(n_embd,head_size,bias=False)
        self.value=nn.Linear(n_embd,head_size,bias=False)
        self.register_buffer('tril',torch.tril(torch.ones(block_size,block_size)))

        self.dropout=nn.Dropout(dropout)
    def forward(self,x):
        B,T,C=x.shape
        k=self.key(x)
        q=self.query(x)

        wei=q@k.transpose(-2,-1) * (C**-0.5)
        #wei=wei.masked_fill(self.tril[:T,:T]==0,float('-inf')) 这一行之后会进行论述
        wei=F.softmax(wei,dim=-1)
        wei=self.dropout(wei)
        v=self.value(x)
        out=wei@v
        return out
```



## 多头attention机制

单头的attention可以提取到某一个方面的信息，并且进行整合输出，但是我们想要提取到不同方面的信息，于是可以借鉴**GoogleNet以及CNN卷积核**的想法，将**多个东西给拍到一起**。具体实现就是将几个attention的输出的head直接连起来。

数学形式为
$$
MultiHead(Q,K,V)=Concat(head_1,head_2,...head_n)W^O\\
head_i=Attention(QW_i^Q,K W_i^K,V W_i^V)
$$
![image-20251015141417392](D:\wesleyfei1.github.io\blogs\Sophomore-Fall\知乎文章\论文阅读1.assets\image-20251015141417392.png)

Pytorch实现就是

```python
class MultiHeadAttention(nn.Module):
    def __init__(self,num_heads,head_size):
        super().__init__()
        self.heads=nn.ModuleList([Head(head_size) for _ in range(num_heads)])
        self.proj=nn.Linear(n_embd,n_embd)
        self.dropout=nn.Dropout(dropout)
    def forward(self,x):
        out= torch.cat([h(x) for h in self.heads],dim=-1)
        out=self.proj(out)
        out=self.dropout(out)
        return out
```



### Transformer中用到的几种注意力机制

**Self-attention**:**Q,K,V**全部来自于一个输入x，用于Encoder阶段，**方便对于输入信息进行整合**

**Cross-attention**:用于Decoder,**Q,K**来自于Encoder的输出结果，而**V**来自于**已经输出的y**,方便**将语义理解按照已经输出的信息的格式，输出下一个结果**

**Masked-attention**:用于Decoder，Q,K,V都来自于outpurs的y，但是对于attention中的$Attention(Q,K,V)=softmax(\dfrac{QK^T}{\sqrt{d_k}})V$

只取$QK^T$的下三角部分，原因是我们对于一个一个词语蹦出来的时候，只能通过已经输出的信息去理解一个东西，而使用下三角矩阵的乘法就可以实现这一点。

具体就是

```python
wei=wei.masked_fill(self.tril[:T,:T]==0,float('-inf'))
```

## 前向传播层(FFN)

有了attention的结果之后我们需要对于已有信息进行更深层次的理解，Google使用了单一隐藏层的两层的网络实现这一点

$FFn(x)=W_2ReLU(W_1 X+b_1)+b_2$

pytorch代码就是

```python
class FeedForward(nn.Module):
    def __init__(self,n_embd):
        super().__init__()
        self.net=nn.Sequential(
            nn.Linear(n_embd,n_embd*4),
            nn.ReLU(),
            nn.Linear(4*n_embd,n_embd),
            nn.Dropout(dropout),
        )
    def forward(self,x):
        return self.net(x)

```



### Transformer中使用的一些regularization技巧

- dropout层，对于每一个head,ffn部分，都使用dropout来减小过拟合(这一点在之后的网络中使用较少)
- 残差连接：还是喜欢ResNets这个名字，图示法就是



因此对于输出，我们使用$x+Attention(x),x+Layer(x)$的方法

- LayerNorm:比如Attention(x)之后的结果是为$[x_1,x_2,...x_n]$，我们通过计算$E(x),Var(x)$进行归一化处理，将数据变成方差1，均值0来减小过拟合趋势

将以上的东西全部合起来，**layernorm数据，经过attention处理以及Resnets连接，FFn整合**形成一个模块，

```python
class Block(nn.Module):
    def __init__(self,n_embd,n_head):
        super().__init__()
        head_size=n_embd//n_head
        self.sa=MultiHeadAttention(n_head,head_size)
        self.ffwd=FeedForward(n_embd)
        self.ln1=nn.LayerNorm(n_embd)
        self.ln2=nn.LayerNorm(n_embd)
    def forward(self,x):
        x=x+self.sa(self.ln1(x))
        x=x+self.ffwd(self.ln2(x))
        return x
```

## 简化Transformer实现

以下代码根据是YouTube大佬，tesla&openai的karpathy的视频实现的https://www.youtube.com/watch?v=kCc8FmEb1nY

我们实现了一个只有decoder的transformer，目的是模仿莎士比亚的文档进行类似风格的创作。其代码如下

```python
import torch
import torch.nn as nn
from torch.nn import functional as F
import time
#hyperparameters
batch_size=64
block_size=256
max_iters=5000
eval_interval=500
dropout=0.2
n_layer=6
n_head=6
learning_rate=3e-4
device='xpu' if torch.xpu.is_available() else 'cpu'

print(device)
eval_iters=200
n_embd=384
#---------------

torch.manual_seed(1337)
with open('karpathy/buildGPT from scratch/input.txt','r',encoding='utf-8') as f:
    text=f.read()

# here are all the unique characters that occur in the text
chars=sorted(list(set(text)))
vocab_size=len(chars)
# create a encoder and a decoder to map from characters to integers
stoi={ch:i for i,ch in enumerate(chars)}
itos={i:ch for i, ch in enumerate(chars)}
encode=lambda s:[stoi[c] for c in s]
decode=lambda l:''.join([itos[i] for i in l])

# train and test splits
data=torch.tensor(encode(text),dtype=torch.long)
n=int(0.9*len(data))
train_data=data[:n]
val_data=data[n:]#to avoid be given stored away given text

# data loading
def get_batch(split):
    # generate a small batch of data of inputs x and targets y
    data=train_data if split == 'train' else val_data
    ix=torch.randint(len(data)-block_size,(batch_size,))
    x=torch.stack([data[i:i+block_size] for i in ix])
    y=torch.stack([data[i+1:i+block_size+1] for i in ix])
    #x,y=x.to(device),y.to(device)
    return x,y

@torch.no_grad()
def estimate_loss():
    out={}
    model.eval()
    for split in ['train','val']:
        losses=torch.zeros(eval_iters)
        for k in range(eval_iters):
            X,Y=get_batch(split)
            logits,loss=model(X,Y)
            losses[k]=loss.item()
        out[split]=losses.mean()
    model.train()
    return out

class BigramLanguageModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.token_embedding_table=nn.Embedding(vocab_size,n_embd)
        self.position_embedding_table=nn.Embedding(block_size,n_embd)
        self.blocks=nn.Sequential(*[Block(n_embd,n_head=n_head) for _ in range(n_layer)])
        self.ln_f=nn.LayerNorm(n_embd)
        self.lm_head=nn.Linear(n_embd,vocab_size)
    def forward(self,idx,targets=None):
        B,T=idx.shape
        #idx and targets are both (B,T) tensor of integers
        pos_emb=self.position_embedding_table(torch.arange(T,device=device))# (T,C)
        tok_emb=self.token_embedding_table(idx)#(B,T,C)
        x=tok_emb+pos_emb# token embedding & positional embedding
        x=self.blocks(x)
        x=self.ln_f(x)
        logits=self.lm_head(x)# (B,T,C)

        if targets is None:
            loss=None
        else:
            B,T,C=logits.shape
            logits=logits.view(B*T,C)
            targets=targets.view(B*T)
            loss=F.cross_entropy(logits,targets)
        return logits,loss
    def generate(self,idx,max_new_tokens):
        # idx is (B,T) array of indices in the current context
        for _ in range(max_new_tokens):
            idx_cond=idx[:,-block_size:]# crop the context

            logits,loss=self(idx_cond)
            logits=logits[:,-1,:]#(B,C)
            probs=F.softmax(logits,dim=-1)
            idx_next=torch.multinomial(probs,num_samples=1)
            idx=torch.cat((idx,idx_next),dim=1)
        return idx
    


model=BigramLanguageModel()
#model = torch.compile(model)
m=model.to(device)
# create a Pytorch optimizer
optimizer=torch.optim.AdamW(m.parameters(),lr=1e-3)


train_data = train_data.to(device)
val_data = val_data.to(device)
start=time.time()
print(sum(p.numel() for p in m.parameters())/1e6,'M parameters')
for iter in range(max_iters):
    if iter%eval_interval==0:
        now=time.time()
        print(now-start)
        losses=estimate_loss()
        print("Memory allocated: {:.2f} MB".format(torch.xpu.memory_allocated(device)/1e6))
        print("Memory reserved: {:.2f} MB".format(torch.xpu.memory_reserved(device)/1e6))
        print(f"step{iter}: train loss{losses['train']:.4f},val loss{losses['val']:.4f}")
    xb,yb=get_batch('train')

    logits,loss=m(xb,yb)
    optimizer.zero_grad(set_to_none=True)
    loss.backward()
    optimizer.step()

#generate from the model
context=torch.zeros((1,1),dtype=torch.long,device=device)
print(decode(m.generate(context,max_new_tokens=500)[0].tolist()))
```
